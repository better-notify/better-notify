> **Status:** ✅ COMPLETE (2026-04-26). All 5 tasks shipped; 268 core tests + 6 smtp tests + 11 typecheck tasks passing. Working tree only — pending commit decision from user. See "Work landed beyond this plan" at bottom.

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build first-class internal observability into `@emailrpc/core` via a `LoggerLike` contract that core uses at every send-pipeline phase, with a default console adapter and BYO-logger support.

**Architecture:** New `packages/core/src/logger.ts` defines `LoggerLike`, `consoleLogger`, `fromPino`. `createClient` accepts `logger?` and threads a per-send child logger (`{ component, route, messageId }`) through the existing `runSend` pipeline in `client.ts`. Errors are normalized to the `err` payload key for pino-stdSerializers compatibility. The existing stub `loggerMw` (middleware.ts) and `loggerPlugin` (plugins/logger.ts) are deleted; logging is no longer opt-in.

**Tech Stack:** TypeScript, vitest, rolldown. No new runtime deps.

**Spec:** `docs/superpowers/specs/2026-04-25-logger-design.md`

```ts
export type LogLevel = 'debug' | 'info' | 'warn' | 'error';

export type LoggerLike = {
  debug(message: string, payload?: object): void;
  info(message: string, payload?: object): void;
  warn(message: string, payload?: object): void;
  error(message: string, payload?: object): void;
  child(bindings: object): LoggerLike;
};

export type ConsoleLoggerOptions = {
  level?: LogLevel;
};

const LEVEL_ORDER: Record<LogLevel, number> = { debug: 10, info: 20, warn: 30, error: 40 };

const serializeError = (e: unknown): unknown => {
  if (!(e instanceof Error)) return e;
  return {
    type: e.name,
    message: e.message,
    stack: e.stack,
    ...(e.cause !== undefined ? { cause: serializeError(e.cause) } : {}),
    ...((e as { code?: unknown }).code !== undefined ? { code: (e as { code: unknown }).code } : {}),
  };
};

const normalizePayload = (payload: object | undefined): object => {
  if (!payload) return {};
  const out: Record<string, unknown> = { ...(payload as Record<string, unknown>) };
  if (out.err !== undefined) out.err = serializeError(out.err);
  return out;
};

const make = (level: LogLevel, bindings: object): LoggerLike => {
  const threshold = LEVEL_ORDER[level];
  const emit = (lvl: LogLevel, sink: (m: string, p: object) => void) =>
    (msg: string, payload?: object): void => {
      if (LEVEL_ORDER[lvl] < threshold) return;
      sink(msg, { ...bindings, ...normalizePayload(payload) });
    };
  return {
    debug: emit('debug', (m, p) => console.debug(m, p)),
    info: emit('info', (m, p) => console.info(m, p)),
    warn: emit('warn', (m, p) => console.warn(m, p)),
    error: emit('error', (m, p) => console.error(m, p)),
    child(extra) {
      return make(level, { ...bindings, ...extra });
    },
  };
};

export const consoleLogger = (opts?: ConsoleLoggerOptions): LoggerLike => {
  return make(opts?.level ?? 'warn', {});
};

type PinoLike = {
  debug(obj: object, msg: string): void;
  info(obj: object, msg: string): void;
  warn(obj: object, msg: string): void;
  error(obj: object, msg: string): void;
  child(bindings: object): PinoLike;
};

export const fromPino = (pino: PinoLike): LoggerLike => ({
  debug: (msg, payload) => pino.debug(payload ?? {}, msg),
  info: (msg, payload) => pino.info(payload ?? {}, msg),
  warn: (msg, payload) => pino.warn(payload ?? {}, msg),
  error: (msg, payload) => pino.error(payload ?? {}, msg),
  child: (bindings) => fromPino(pino.child(bindings)),
});
```

```ts
import type { LoggerLike, LogLevel } from './logger.js';

export type LogRecord = {
  level: LogLevel;
  message: string;
  bindings: Record<string, unknown>;
  payload: Record<string, unknown>;
};

export type MemoryLogger = LoggerLike & {
  readonly records: ReadonlyArray<LogRecord>;
  clear(): void;
};

const buildMemoryLogger = (
  records: LogRecord[],
  bindings: Record<string, unknown>,
): MemoryLogger => {
  const push = (level: LogLevel) =>
    (message: string, payload?: object): void => {
      records.push({
        level,
        message,
        bindings: { ...bindings },
        payload: { ...((payload as Record<string, unknown>) ?? {}) },
      });
    };
  return {
    get records() {
      return records;
    },
    clear() {
      records.length = 0;
    },
    debug: push('debug'),
    info: push('info'),
    warn: push('warn'),
    error: push('error'),
    child(extra) {
      return buildMemoryLogger(records, { ...bindings, ...(extra as Record<string, unknown>) });
    },
  };
};

export const memoryLogger = (): MemoryLogger => buildMemoryLogger([], {});
```

- `consoleLogger` got a pino-pretty-style formatter: `[time] LEVEL (component/route): message` + indented `key: value` payload lines. Color via TTY detection. Errors render with type/message/stack/cause/code. `durationMs` rendered as `0.7ms`; `messageId` truncated to last 8 chars; `undefined` fields stripped.
- Coverage: `logger.ts` at 100/98.75/100/100 (only `process.stdout?.isTTY` branch unmockable).

**Folder reorganization (matches CLAUDE.md convention)**
- `middlewares/` folder: per-`withName` files + `<name>.types.ts` only when the middleware has its own options + shared `types.ts` + `index.ts` barrel. All `*Mw` factories renamed to `withX`: `dryRunMw → withDryRun`, `tagInjectMw → withTagInject`, etc.
- `plugins/` folder: same pattern (`types.ts` + `index.ts`).
- `transports/` folder: same pattern.

**Provider → Transport rename**
- `Provider` type → `Transport`, `ProviderEntry` → `TransportEntry`, `mockProvider` → `mockTransport`, `multi` → `multiTransport`.
- `createClient({ providers: [...] })` → `createClient({ transports: [...] })`.
- Logger payloads renamed: `providerName` → `transportName`, `providerMessageId` → `transportMessageId`.
- The unimplemented `smtp()` stub removed from core (moved to its own package — see below).

**Address shape rename**
- `Address = string | { name?: string; email: string }` (was `address`).
- New `FromInput = string | { name?: string; email?: string }` (both optional). `.from()` and `defaults.from` accept `FromInput`; `resolveFrom()` in `client.ts` shallow-merges per-field at send time.

**`./test` subpath removed**
- `mockTransport` promoted to public `transports` surface (useful dev tool).
- `memoryLogger` and `recordHooks` stay in `packages/core/src/test.ts` as **internal-only** — used by core's own tests, not built into `dist`, not exported via subpath.

**New package: `@emailrpc/smtp`**
- Real nodemailer-backed `smtpTransport(opts)` returning `Transport`.
- One-time `warn` log per unique mismatching from-address (when `auth.user` is set and message `from.email` differs).
- README covers options, sending-limits guidance, from-rewriting caveat, sender-verification expectations.
- 6 tests, 100% coverage.

**Shared transport utilities**
- `formatAddress` (RFC 5322 `"Name" <email>`) and `normalizeAddress` (bare email) live in `packages/core/src/transports/utils.ts` and are re-exported from `@emailrpc/core/transports`. Used by both `mockTransport` and `smtpTransport` (no duplication across packages).
- JSDoc on the public `Transport` contract surface only — no body comments — so future adapter implementers know the contract.

**Lint rules**
- `typescript/no-non-null-assertion`, `no-console`, `no-else-return` (with `allowElseIf: false`) all `error`.
- Overrides: tests, `logger.ts`, and `examples/**/*.ts`.
- All 9 pre-existing `!` in `client.ts` refactored to proper narrowing (handlePromise-tuple-by-index pattern + array-index guards).

**Example app** (`examples/welcome-text/apps/cli`)
- Wired up to real SMTP via `dotenvx` + `@t3-oss/env-core`. Sends a real email end-to-end with the new pretty logger output.
- Uses `defaults.from: { name: env.SMTP_FROM_NAME, email: env.SMTP_USER }`.
