Build custom transports with the step-by-step guide in [Generic Custom Transports](/docs/transports/generic/custom-transports).

If you're wiring an email provider, start there and import from `@betternotify/email/transports`. Use `@betternotify/core/transports` only when you're building channel-agnostic helpers.
