# Product

## Register

product

## Users

Backend and fullstack engineers building notification infrastructure in Node.js applications. They know what they want, scan for specifics, and judge a tool by how quickly they can understand its behavior. They read docs on a laptop, often with an IDE open beside the browser.

## Product Purpose

Better-Notify is an end-to-end typed notification infrastructure for Node.js. The documentation site teaches engineers how to configure and compose the library's primitives (transports, middleware, queues, webhooks) through interactive examples that show behavior, not just API signatures.

## Brand Personality

Technical. Precise. Unadorned.

## Anti-references

- Generic documentation sites with placeholder content and default themes
- Cookie-cutter API reference layouts that all look the same
- Overdesigned marketing-heavy docs that prioritize style over substance

## Design Principles

1. **Show behavior, not just signatures.** Interactive components that demonstrate how things work replace walls of prose.
2. **Respect the engineer's time.** Every element earns its space. No filler, no decoration for its own sake.
3. **Technical confidence.** The interface should feel like it was built by someone who uses tools like this, not someone selling them.
4. **Reusable patterns.** Interactive components should be composable and adaptable across different documentation contexts.

## Accessibility & Inclusion

WCAG AA. Respect prefers-reduced-motion for all animations. Ensure signal flow states are distinguishable without color alone.
